Finals Lab Task 5

SQL Views, Stored Procedures, and Functions

Project Overview

a In this portfolio, I showcase how I used SQL views, stored procedures, and functions to handle and work with database data. I included examples like using views to filter information, stored procedures to update records, and functions to pull specific data.

Step by Step Process

Step 1: Create a View for Vendors and Products (2002+)

  • Display vendors_code, vendors name, product description, and p_indate for all products with p_indate from 2002 onwards.

Step 2: Create a View for Products in Price Range 100-150

  • Display all products whose price range is between 100-150.

Step 3: Create a View for Total Price by Vendor

  • Compute the TOTAL_PRICE of ALL PRODUCTS by getting (P_ONHAND x P_PRICE) sold by vendors with v_code 21344, 23119, and 24288.

Step 4: Stored Procedure to Update Vendor Name

  • Create a stored procedure that takes a single parameter and updates the name of vendor 'Bryson,Inc.' to 'Bryson and Co'.

Step 5: Function to Display Products by Vendor and State

  • Create a function that takes 2 parameters (v_code and v_state) and displays all the product description and price based on the parameters passed to the function.
